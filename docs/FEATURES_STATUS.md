# حالة الميزات — المشروع مع Supabase

مراجعة ميزة ميزة بعد ربط قاعدة البيانات بـ Supabase.

---

## ✅ جاهز (السكيما + الكود مربوط بالـ DB)

| الميزة | الجداول | ماذا يعمل |
|--------|---------|-----------|
| **تسجيل الدخول / الحساب** | `user`, `session`, `account`, `verification` | Better Auth — تسجيل دخول (بريد/كلمة مرور + جوجل)، جلسات، ربط حسابات. |
| **الأسئلة الأولية (Onboarding)** | `user` | إكمال البروفايل (جنس، عمر، طول، وزن، نشاط، هدف) وحساب الهدف اليومي من السعرات. |
| **لوحة التحكم الرئيسية** | `user`, `calorie_entry` | عرض الاسم، الهدف اليومي، وإجمالي السعرات المستهلكة اليوم من `calorie_entry`. |
| **السعرات اليومية** | `calorie_entry` | إضافة مدخلات سعرات يدوية (`addCalorieEntry`) وعرض ملخص اليوم (`getTodayCalorieSummary`). الداشبورد يعرض المجموع عبر `getDashboardSnapshot`. |
| **نظام التغذية / تحليل الوجبات** | `meals` | رفع صورة وجبة → Cloudinary، تحليل بـ Gemini، حفظ في `meals`. عرض قائمة الوجبات مع فلترة بالتاريخ. |
| **نظام الكريدت (Polar)** | `user.polar_customer_id` | ربط المستخدم بـ Polar، خصم كريدت عند تحليل وجبة، منح كريدت عند الدفع، Webhook لـ `order.paid`. |

---

## ⚠️ السكيما جاهزة لكن الواجهة لا تستخدم DB بعد

| الميزة | الجداول | الوضع الحالي |
|--------|---------|----------------|
| **النشاط البدني (Workout)** | `tasks`, `time_sessions` | الواجهة (Header، WeeklyProgress، WorkoutGroups) تعرض بيانات ثابتة أو وهمية. لا يوجد استدعاء لـ DB لقراءة/كتابة مهام أو جلسات. |
| **العادات والتركيز (Habits)** | `tasks`, `time_sessions` | متتبع العادات (HabitsTracker) والتركيز (FocusTimer) يعتمدان على state محلي فقط. لا حفظ للمهام أو جلسات الوقت في قاعدة البيانات. |

لتفعيلها لاحقاً: إنشاء Server Actions (مثلاً `getTasks`, `addTask`, `getTimeSessions`, `addTimeSession`) وربطها بالمكونات.

---

## 📌 ملاحظات

- **رسم الوزن (Weight chart):** السكيما الحالية تحتوي على `user.weight` (قيمة واحدة فقط). لا يوجد جدول لتاريخ الوزن (مثلاً `weight_entry` مع تاريخ + وزن). لو أردت رسم تطور الوزن مع الزمن، يُفضّل إضافة جدول لتسجيل القيم حسب التاريخ.
- **التغذية:** الوجبات تُخزَّن في `meals`؛ إجمالي السعرات المعروضة في الداشبورد يأتي من `calorie_entry` فقط. إذا أردت دمج سعرات الوجبات المحللة في المجموع اليومي، تحتاج منطق يجمع من `calorie_entry` + سعرات من `meals` لنفس اليوم.

---

## الخلاصة

- **جاهز بالكامل:** Auth، Onboarding، الداشبورد، السعرات اليومية، تحليل الوجبات، Polar.
- **سكيما جاهزة والواجهة تحتاج ربط:** Workout، العادات والتركيز.
- **اختياري لاحقاً:** جدول تاريخ الوزن إذا أردت رسم تطور الوزن.

مع Supabase والـ schema الحالية، الميزات الجاهزة في الجدول الأول تعمل دون تغيير إضافي على السكيما.
